奶头乐

tittytainment战略，又叫奶头乐战略。

1995年9月27日到10月1日，美国旧金山集合世界500多名政治经济精英（包括撒切尔、老布什等），讨论如何应对全球化。

全球化加速贫富差距，财富金重在20%手上，80%被边缘化，如何化解冲突？消解80%人口的多余精力和不满情绪，转移他们的注意力？

美国高级智囊布热津斯基认为，唯一的方法是给80%的人口塞上一个奶嘴。让他们安于为他们量身定制的娱乐信息中，慢慢丧失热情、抗争欲望和思考能力。

这就是闻名遐迩的Tittytainment战略，由titty（奶嘴）与Entertainment（娱乐）合成，中文译为奶透了战略。

头号玩家

<头好玩家> 正是奶头乐的一种：轻而易举获得巨大成功。

巨量的宝藏，千万人智慧与心血的结晶，全人类的安全，都在此一搏。反派装备精良人手众多却通通蠢得要命，只要主角下定决心，灵光一闪，倒着开车，邀请女孩子跳舞，玩个老游戏，所有荣耀与财富全都归他支配。

一类人和另一类人

芸姨的影评把《头号玩家》的观众分成2种：屌丝和大哥。

屌丝与大哥脑回路不同。
屌丝站在消费者的角度，渴望被理解、被认可。大哥站在供应商的角度，分析观众的g点，下海实践创造。

屌丝与大哥脑容量相似。但区别在于，屌丝在幻觉和情绪中消磨自己。大哥深知脑容量是稀缺的，不能浪费在不产生效益的事情上。

稀缺

宇宙每时每刻在发生兑换。有人用时间兑换了幻觉，有人用金钱兑换了食物。有人用精力兑换了荣耀，有人用荣耀兑换了金钱……我之前意识不到，但其实我一直都是生意人。

粗心大意的生意人，把好东西兑换成辣鸡，势力越来越弱。而精明的生意人会把生意越做越大，把手头的资源兑换成价值更大的资源。

所以要搞清楚自己手上的资源的价值。对于稀缺的资源，配置要更加谨慎。

wojiubuda 2017-07-23

本篇目的：工作准备，中吉综合机的逻辑、前端、后台、手机端，咖啡机的逻辑、维护端。

中吉综合机的功能点

PC后台：
  1. 给机器的每个货道配货、换货、合并货道。货道很难拆下来，怎么合并
售货机端：
  1. 同步商品信息：商品编号、名称、图片、价格、数量。永久化信息到本地。
  2. 售卖：选货道进入商品详情
    • 进入详情前：判断货道是否可售（可售定义如下)，如果可售，则查询所选货道的商品信息。
      • 货道无故障 能否出货前查询所选货道是否故障
      • 货道有库存
      • 货道无连续2次出货失败记录
    • 收到订单已支付消息后，提示支付成功。
    • 出货中：检测出货是否成功，展示状态（出货中，出货成功/出货失败，请联系客服）。
    • 出货后：
      • 出货成功：更改该货道库存，缓存订单编号 为什么 上报出货成功状态，清除上次出货失败记录。
      • 出货失败：上报出货失败，存储出货故障。
  3. 故障查询与上报 是否有自动查询故障的机制
  4. 维护模式
    • 纸币可以退回、硬币可以找零、不买货可以找零。不用显示在屏幕上。
    • 遥控出货开关
    • LED灯开关
    • 温度开关：1区2区是什么，设置温度有什么用，温度上限和下限有什么用
    • 制冷加热：（只要开制冷，自动开启玻璃除雾）
    • 钱币预装： 点了有什么用
    • 清除故障：有什么用
    • 测试货道出货
  5. 广告、取货码

缓缓吸气，感受空气的讯息，让好闻的味道和自己融为一体。

昨天看了一部电影《香水》。

这部分是剧透，介意勿看。

变态男主一生对味道痴迷，他创造的香水配方风靡巴黎，让门可罗雀的香水铺子变得人满为患。

变态男主成年后发现，最好闻的味道，是年轻貌美的姑娘的味道，想永久保存。于是潜心学会味道提取术，谋杀了多个姑娘，最终制成了极其好闻的香水。

有多好闻呢？ 男主被判死刑，行刑前，滴几滴香水在手绢上，挥向看行刑的人群，众人波浪般倾倒，如痴如醉。然后都给他跪下，为什么呢，太好闻了，不可能是坏人。大家一致同意放走他。

售货机的逻辑

wojiubuda 2017-07-19

写此篇的目的

我们要卖中吉的综合机，软件需要我们自己写。所以我们要根据中吉给的底层协议，来实现售货机的功能。

售货机需要哪些功能？

共性功能：

  1. 初始化：导入初始相关数据，使其可售卖。
    • 条件：[售货机端请求]时
    • 内容：每个货道对应的商品信息：图片、中英文名称、价格、库存。
  2. 同步更新：后台更新的数据，同步到售货机端。
    • 条件：[机器重启]时
    • 内容：商品图片、中英文名称、价格、库存。
  3. 售卖：展示支付二维码前查询可售状态、收到支付成功的推送后出货、收到出货结果后上报且更新可售状态。
    • 展示支付二维码前查询可售状态
      • 条件：用户输入货道号时
      • 查询内容
        • 货道是否故障
        • 商品是否售空
        • 是否连续2次出货失败历史
    • 收到支付成功的推送后 出货
      • 展示状态：出货中、出货成功、出货失败。
    • 收到出货结果后
      • 出货成功：更改该货道库存，上报出货成功状态，清楚上次出货失败记录。
      • 出货失败：上报出货失败状态，存储此次出货失败记录。
  4. 故障查询：定时、特定触发条件查询机器故障。
    • 查询内容：暂无货道故障以外的故障，需要电话问。
  5. 故障上报：定时、或特定触发条件上报机器故障。
  6. 心跳：监控网络状态。

个性功能

  1. 性能测试
    • 调货检测灵敏度
  2. 设置
    • 制冷加热系统：制冷 加热 关闭
    • 设置温度上下限
    • 玻璃除雾：开关
